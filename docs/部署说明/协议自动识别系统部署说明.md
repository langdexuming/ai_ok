## 1 系统概述

协议自动识别系统分为两个子项目：
- **云端OMC工具**：部署在云端服务器，用于管理识别库、校核配置和设备
- **边端批量配置工具**：部署在边缘网关或本地PC，用于协议自动识别

## 3 云端OMC工具部署

### 3.1 数据库初始化

```bash
# 登录MySQL
mysql -u root -p

# 执行初始化脚本
source /path/to/omc-cloud/omc-backend/src/main/resources/db/init.sql
```

或者直接执行：

```bash
mysql -u root -p < omc-cloud/omc-backend/src/main/resources/db/init.sql
```

### 3.2 后端部署

#### 3.2.1 编译

```bash
cd omc-cloud/omc-backend
mvn clean package -DskipTests
```

编译完成后在 `target/` 目录生成 `omc-backend-1.0.0.jar`。

#### 3.2.2 配置修改

编辑 `application.yml` 或通过环境变量覆盖：

```yaml
spring:
  datasource:
    url: jdbc:mysql://数据库地址:3306/pai?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    username: 数据库用户名
    password: 数据库密码

omc:
  upload:
    path: /data/omc/upload    # 动态库文件存储路径
  sync:
    interval-days: 10          # 设备配置同步间隔(天)
```

#### 3.2.3 启动

```bash
# 前台启动
java -jar target/omc-backend-1.0.0.jar

# 后台启动
nohup java -jar target/omc-backend-1.0.0.jar > /var/log/omc-backend.log 2>&1 &

# 指定配置文件
java -jar target/omc-backend-1.0.0.jar --spring.config.location=classpath:application.yml,file:/etc/omc/application.yml
```

#### 3.2.4 Systemd服务（推荐）

创建 `/etc/systemd/system/omc-backend.service`：

```ini
[Unit]
Description=OMC Backend Service
After=network.target mysql.service

[Service]
Type=simple
User=omc
WorkingDirectory=/opt/omc
ExecStart=/usr/bin/java -jar /opt/omc/omc-backend-1.0.0.jar
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable omc-backend
sudo systemctl start omc-backend
```

### 3.3 前端部署

#### 3.3.1 编译

```bash
cd omc-cloud/omc-frontend
npm install
npm run build
```

编译完成后在 `dist/` 目录生成静态文件。

#### 3.3.2 Nginx配置

```nginx
server {
    listen 80;
    server_name omc.example.com;

    root /opt/omc/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:9000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
        client_max_body_size 100m;
    }
}
```

```bash
sudo nginx -t
sudo nginx -s reload
```

### 3.4 验证

1. 访问 `http://omc.example.com`，确认页面正常加载
2. 查看API文档：`http://omc.example.com:9000/doc.html`
3. 检查后端日志：`journalctl -u omc-backend -f`

## 4 边端批量配置工具部署

### 4.1 后端部署

#### 4.1.1 编译

```bash
cd edge-tool/edge-backend

# 编译当前平台
go build -o edge-backend

# 交叉编译ARM Linux
GOOS=linux GOARCH=arm64 go build -o edge-backend-arm64

# 交叉编译x86 Linux
GOOS=linux GOARCH=amd64 go build -o edge-backend-amd64
```

#### 4.1.2 配置

通过环境变量配置：

```bash
export EDGE_PORT=9090          # 服务端口
export EDGE_DB_PATH=./i-edge.db  # SQLite数据库路径
```

#### 4.1.3 部署到边缘网关

```bash
# 拷贝可执行文件和数据库
scp edge-backend-arm64 root@边缘网关IP:/opt/edge/edge-backend
scp i-edge.db root@边缘网关IP:/opt/edge/i-edge.db

# 登录设备启动
ssh root@边缘网关IP
cd /opt/edge
chmod +x edge-backend
./edge-backend
```

#### 4.1.4 Systemd服务

创建 `/etc/systemd/system/edge-backend.service`：

```ini
[Unit]
Description=Edge Backend Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/edge
Environment=EDGE_PORT=9090
Environment=EDGE_DB_PATH=/opt/edge/i-edge.db
ExecStart=/opt/edge/edge-backend
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 4.2 前端部署

#### 4.2.1 编译

```bash
cd edge-tool/edge-frontend
npm install
npm run build
```

#### 4.2.2 Nginx配置

```nginx
server {
    listen 80;
    server_name localhost;

    root /opt/edge/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:9090/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }
}
```

### 4.3 验证

1. 访问 `http://边缘网关IP`
2. 检查后端日志：`journalctl -u edge-backend -f`

## 6 常见问题

### 6.1 数据库连接失败

- 检查MySQL服务是否启动：`systemctl status mysql`
- 检查数据库连接配置（地址、端口、用户名、密码）
- 确认pai数据库是否已创建

### 6.2 端口被占用

```bash
# 查看端口占用
netstat -tlnp | grep 9000
lsof -i:9000

# 修改端口
# Java: 修改application.yml中server.port
# Go: 设置环境变量EDGE_PORT
```

### 6.3 文件上传失败

- 检查上传目录权限：`chmod -R 755 /data/omc/upload`
- 检查磁盘空间：`df -h`
- 检查Nginx配置中的`client_max_body_size`

### 6.4 边端无法连接OMC

- 检查网络连通性：`ping OMC_IP`
- 检查OMC端口是否开放：`telnet OMC_IP 9000`
- 检查防火墙规则

### 6.5 协议识别失败

- 检查识别库数据是否已导入
- 检查校核配置的上下限设置是否合理
- 查看结果打印日志中的失败原因

## 7 版本更新

### 7.1 后端更新

```bash
# 停止服务
sudo systemctl stop omc-backend

# 替换jar包
cp omc-backend-1.0.0.jar /opt/omc/

# 启动服务
sudo systemctl start omc-backend
```

### 7.2 前端更新

```bash
# 替换dist目录
rm -rf /opt/omc/frontend/dist
cp -r dist/ /opt/omc/frontend/dist

# 重载Nginx
sudo nginx -s reload
```

### 7.3 数据库升级

- 备份数据：`mysqldump -u root -p pai > pai_backup.sql`
- 执行升级脚本
- 验证数据完整性
