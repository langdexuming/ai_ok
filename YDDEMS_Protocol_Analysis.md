## 2. 设备类型说明

YD-DEMS-BD2 是一款交流配电系统监控设备，支持：
- **2路进线** (主电源输入)，每路均可采集三相电压、电流、功率、功率因数、频率、电能、谐波等完整电量参数
- **144路馈出回路** (支路)，每路可采集电压、电流、有功功率、有功电能、无功功率、视在功率、功率因数、谐波相关参数
- **144路温度** 监测点
- **144路支路开关** 状态及告警
- **主路开关** 状态（含防雷器开关、备用开关等）
- **阈值设置** (分路电流上上限/上限/下下限/下限)
- **谐波分析** (基波、谐波、畸变率等)

## 3. 通信协议详解

### 3.1 Modbus RTU 帧格式

**请求帧（读保持寄存器 FC=0x03）:**

| 字段 | 字节数 | 说明 |
|---|---|---|
| 从站地址 | 1 | 固定 0x01（硬编码） |
| 功能码 | 1 | 0x03 (读保持寄存器) |
| 起始地址H | 1 | 寄存器起始地址高字节 |
| 起始地址L | 1 | 寄存器起始地址低字节 |
| 寄存器数量H | 1 | 读取数量高字节 |
| 寄存器数量L | 1 | 读取数量低字节 |
| CRC16 L | 1 | CRC低字节 |
| CRC16 H | 1 | CRC高字节 |

**写入帧（FC=0x06 / 0x10）:**
使用 `PackCmdWrite10` 方法封装，支持功能码 0x06 (写单个寄存器) 用于阈值设置。

**响应帧:**

| 字段 | 字节数 | 说明 |
|---|---|---|
| 从站地址 | 1 | 0x01 |
| 功能码 | 1 | 0x03 |
| 数据长度 | 1 | 后续数据字节数 |
| 数据区 | N | 寄存器数据 |
| CRC16 L | 1 | CRC低字节 |
| CRC16 H | 1 | CRC高字节 |

### 3.2 数据类型

- **浮点数 (Float)**: 4字节，低WORD在前，高WORD在后（Low-Word First）。解析方式：先读2个WORD（各自高字节在前），组合为 `(wValueH << 16) + wValueL`，再 `memcpy` 为 float。
- **有符号短整型 (Short)**: 2字节，高字节在前，用于功率因数（*0.001）和总谐波畸变率（*0.01）。
- **无符号短整型 (WORD)**: 2字节，高字节在前，用于温度（*0.1）和部分谐波数据（*0.01）。
- **位域 (Bit)**: 单WORD的各位表示不同告警/开关状态。

## 6. 数据解析逻辑详解

### 6.1 浮点数解析（进线数据、馈出回路数据、电能数据）

```cpp
safeBuf.ReadWord(wValueL, HighFirst);  // 读低WORD
safeBuf.ReadWord(wValueH, HighFirst);  // 读高WORD
dwValue = (wValueH << 16) + wValueL;   // 组合为DWORD
memcpy(&fValue, &dwValue, sizeof(float)); // 按IEEE754解释为float
```

**字节序说明**: 每个WORD内部高字节在前(Big-Endian)，但两个WORD组合时低WORD在前(Low-Word First)。这是 Modbus 浮点数常见的 "CDAB" 字节序。

### 6.2 温度解析

```cpp
safeBuf.ReadWord(wValue, HighFirst);
fValue = wValue * 0.1;  // 单位：0.1度
```

### 6.3 告警位解析

```cpp
safeBuf.ReadWord(wValue, HighFirst);
for (int i = 0; i < 16; i++) {
    byValue = GETBIT(wValue, i);  // 逐位提取
    OUTPUT_VALUE(baseID + i, byValue);
}
```

### 6.4 开关状态解析

```cpp
byValue = GETBIT(wValue, j);
byValue = (byValue == 0) ? 1 : 0;  // 取反: 0=启用, 1=不启用
```

### 6.5 停电/缺相判断逻辑

```cpp
// 遍历A/B/C三相电压
for (int i = 0; i < 3; i++) {
    if (fValABC[i] < 100) nQXtotal++;  // 电压<100V视为缺相
}
// nQXtotal=1: 缺一相, =2: 缺二相, =3: 停电
```

### 6.6 总输入有功电能计算

```cpp
// CMD1 解析时记录: m_totalPower = 1路正向有功电能(1041)
// CMD2 解析时累加: m_totalPower += 2路正向有功电能(2041)

```

### 6.7 回路所有数据轮询(CMD 23-54)

使用 `m_byNum` 成员变量作为回路计数器，每次调用 PackCmdPowerPartDataALL 时自增，范围 1-50，循环复位。寄存器地址计算：`16384 + (m_byNum - 1) * 64`。

解析时使用 `JumpBack` 在数据缓冲区内前后跳转读取不同偏移位置的数据。

## 10. 代码问题分析

### 10.1 严重Bug

#### (1) UnPackCmdSecondFundamental 输出通道错误
```cpp
// YDDEMSCmd.cpp 第1785-1791行
// 读取的是第2路基波数据，但输出到了第1路通道
OUTPUT_VALUE(1300 + i, fValue);  // 应为 OUTPUT_VALUE(2300 + i, fValue)
```
**影响**: 第2路基波数据覆盖第1路数据，第2路基波通道(2300-2326)永远无数据。

#### (2) PackCmdSetData 无效的size检查
```cpp
// YDDEMSCmd.cpp 第1190行
if (InputChannelList.size() < 0)  // size()返回unsigned, 永远>=0, 此条件永假
```
**影响**: 空列表时不会被拦截，后续 `InputChannelList[0]` 将越界访问。

#### (3) PackCmdOnOffStatusData 返回数据长度检查错误
```cpp
// YDDEMSCmd.cpp 第964行
if (byDataLen != 70)  // 请求9个寄存器=18字节, 应为 != 18
```
**影响**: 长度检查不匹配，但因检查块为空（见下一条），实际不影响运行。

### 10.2 中等问题

#### (4) 大量空的数据长度检查块
在几乎所有 UnPack 函数中，`byDataLen` 检查块为空:
```cpp
if (byDataLen != 240)
{
    // 什么都没做！没有return FALSE，没有日志
}
```
**出现位置**: CMD 1-12, 13, 14, 15, 18 的解包函数中（约15处）。
**影响**: 即使收到错误长度的数据也会继续解析，可能导致数据错误或缓冲区越界。

#### (5) m_totalPower 累加依赖命令执行顺序
```cpp
// CMD1: m_totalPower = fValue;  (赋值)
// CMD2: m_totalPower += fValue; (累加)
```
如果CMD2在CMD1之前执行，或CMD2执行多次而CMD1未重新执行，`m_totalPower` 值将错误。

#### (6) 寄存器地址冲突: CMD 22 与 CMD 63
两个命令读取相同的起始地址 1926 (0x0786):
- CMD 22 (进线2谐波数据): 地址 1926, 读取 58 个寄存器
- CMD 63 (读取2路基波数据): 地址 1926, 读取 54 个寄存器

可能是有意共享地址范围，但 CMD 63 的解包函数错误地写入了1路通道(见Bug#1)。

#### (7) m_byNum 轮询计数器与命令ID不匹配
- 注册了32个命令(CMD 23-54)，但 `m_byNum` 循环范围是 1-50
- 所有32个命令共享同一个Pack/Unpack函数，`m_byNum` 的自增不与特定CMD ID绑定

#### (8) PackCmdSetData 使用功能码 0x06 但通过 PackCmdWrite10 发送
`PackCmdWrite10` 方法的帧格式包含"数据区字节数"字段，这是FC=0x10(写多寄存器)的格式。但实际调用时传入FC=0x06（写单寄存器），帧格式不匹配。

### 10.3 代码质量问题

#### (9) 硬编码从站地址
```cpp
m_byAddrCode = 0x01;  // 固定地址, 不可配置
```

#### (10) 变量遮蔽
```cpp
int i = 0;
// ...
for (int i = 0; i < 3; i++)  // 内层i遮蔽外层i
```
出现在 UnPackCmdFirstDataFloat 和 UnPackCmdSecondDataFloat 中。

#### (11) 缺相判断阈值硬编码
```cpp
if (fValABC[i] < 100)  // 100V 硬编码，无法配置
```

#### (12) 大量重复代码
CMD 3-11 的Pack/Unpack函数几乎完全相同，仅起始地址和通道偏移不同。可以通过参数化重构为一个通用函数。

#### (13) 谐波解析中的 JumpBack 使用
```cpp
safeBuf.JumpBack(4);  // 回退4字节
```
在 UnPackCmdFirstDataHarmonic/UnPackCmdSecondDataHarmonic 中，读完"功率基波"后回退4字节再开始读"电压谐波"。这说明数据区有重叠或跳跃，但缺少注释说明原因，增加维护难度。

#### (14) 版本记录未更新
代码版本为 4.0.1.2，但版本修改记录仅有 4.0.1.1 的说明。

#### (15) 未使用的变量
`UnPackCmdFirstDataHarmonic` / `UnPackCmdSecondDataHarmonic` 中声明了 `WORD wValue = 0` 但未使用。`UnPackCmdFirstPower2` / `UnPackCmdSecondPower2` 中同样有未使用的 `wValue`。
